基于香港市场的订餐平台。

项目于2018年7月在香港上线运营至今，接入3000+本地餐厅，在香港拥有40多万用户。

我在项目中主要负责商户端产品设计(APP&PC)、用户研究工作。

​

本篇主要着眼于项目上线初期时间节点、阐述了设计过程中遇到的问题、运用的方法、以及最后如何解决的过程。

主要方法：访谈、可用性测试、问卷。

Challenge

项目在7月份上线运营两个多月后，我们通过后台数据发现一些问题：

  • 存在不少因”商家拒单“产生的交易失败订单。

  • ​用户评价中很多人抱怨”取餐时还要再等待很久“，或者是”餐品已经凉了“

​同时，商务运营团队向我们反馈香港的商家对互联网平台大部分持保守态度，使用意愿并不高。

为了找到问题存在的原因，我组织并主导了本次调研计划。

调研目标：获取商家用户画像以辅助设计&运营策略、对产品主要功能进行可用性测试。

深圳团队

香港团队

Sampling

我们分为3组，分别去到香港3个大区：新界、九龙、港岛进行商户访谈。总共走访了12家餐厅，餐厅类型包含茶餐厅、连锁快餐店、饮品店、西餐厅。

​

之所以这样选择是因为这三个大区人均消费水平不同，其中港岛>九龙>新界。而这几种类型的餐厅在香港比较普遍，不同类型的餐厅在人员结构、运营模式上存在差异。

​

​为了获得商家画像我们以人均消费水平和餐厅类型作为自变量，观察客流量、餐厅成员结构、消费群体、对互联网接受程度等因变量指标。

Interview

调研分为访谈和可用性测试两个部分，我们准备了10个小问题和3个任务给商家。

persona

​横纵轴基于影响商家使用意愿的主要因素。以下为不同画像的典型特征。针对不同的人群，向运营团队提出有针对性的方案

观望开放 稳健进步者

Key beliefs

认为积极地宣传就能给餐厅增加订单量 认为在商场内的餐厅更适合外卖模式 认为香港人不愿意绑信用卡的问题亟待解决

变中求生 迫切进步者

Key beliefs

由于地理位置原因导致客流不足，需要提高订单量 希望通过多个渠道来单

积极大胆 疆域开拓者

Key beliefs

对能够引流的平台都愿意大胆尝试、反正不亏 对在座点餐等新模式抱有较大的兴趣 觉得平台的推广对餐厅是大有裨益的

​沉睡市场 等待激活者

Key beliefs

担心外卖业务会给餐厅带来额外负担和不好的影响 认为客人来到餐厅一般会选择堂食、不会把食物带回家

设计原则：

1、尊重用户已有使用习惯，保证基础操作的流畅性和连贯性          

鉴于已有外卖平台接入已培养形成用户习惯、尽量保持相类似功能操作上的一致性。

​

2、增强文字大小对比，可以适度夸张       

由于很大部分商家年纪比较大，该强提醒的地方需要适度夸张、以免被忽略。

​

​3、简化操作，能选择的就不要输入      

由于POS机界面操作的灵敏度不如手机好，屏幕尺寸也具有限制，所以尽量简化商家在POS机上的操作。

增加了预订单功能，将距离取餐时间较长的订单延后处理，当到达备餐时间，重新进入新订单列表，并伴有强提示，以解决商家提前备餐的问题

before

增大了排单号的显示，并增加了取餐时间倒计时功能，以解决商家不知该何时备餐和老人家视力不佳的问题

3

弱化了“扫一扫”的入口，并在商家教育上鼓励“自己核对、自己拿走”的核销方式，以解决POS机扫码核销后忘记充电的问题

一开始，我们将预订单的入口放在了订单查询模块内，为了不对商家产生打扰，希望商家只需要在新订单tab下进行操作。但在实际商家的操作中，他们并不能理解“订单去了哪里？”于是我们改进了方案，将预订单收纳到了左上角，并加入了一个抛物线动画及toast提示以让商家明白订单的去向。
